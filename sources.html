Developer Resources

Building websites from scratch can be very challenging at times depending on it's complexity. You may be wondering how is it possible that you know the solution to every problem a mock may present you? The answer is, simply, that you can't!

Much like any other subject, it's impossible for any developer to know every thing there is to know about web development. The most important skill of development is knowing how to research and learn new technologies and features as needed. In my learning path, i am coming to realize that DOCUMENTATION (through books and or internet research) and DEBUGGING SKILLS are criticals for solutions to new problems. This material will give you the tools and confidence to find solutions to issues that arise as you build websites.

DOCUMENTATION

Every well-developed language or framework has a place where you can look up all of its features. This library of information is called a language's documentation. Here are a few i have found highly relevant.

Mozilla Developer Network

Referred to as MDN, it contains the complete documentation for HTML, CSS, JavaScript, and many other essential tools of web development. For instance to see the documentation for a specific CSS property, navigate to the MDN website and search for that property in the search bar at the top of the page, then click on the link for that property.

For example, let's search for the font-weight property.Once you land on a property's documentation page you can read all of the information about it. For example, font-weight starts with a description of the property. The "Values" section lists all of the possible values for font-weight. There are some values you already know, such as normal and bold, but there are also new values, such as lighter and bolder.

Google

Now what if you don't even know what property you need? It is not surprising that using Google is a great way to read other developers' solutions to problems. Googling the problem you're trying to solve and the language you're using will return many useful answers.

For example, if you want to know how to change the size of text, you could google "how to change text size using CSS." After doing this, you will receive multiple links to information about the font-size property.

Stack Overflow

Stack Overflow is a useful resource where developers ask and answer questions. I use it frequently to find answers to some questions, and learn from experienced developers.

To search for a question on Stack Overflow, type your question and programming language into the search bar and find the question threads that seem relevant. Reading all of the top-voted answers to a Stack Overflow question can give you multiple solutions to try as well as an understanding of why one solution might be better than another.

For example, i needed to know how to make a button to have rounded edges. In Stack Overflow search box i typed: ..How to make buttons round using CSS.. The most relevant suggestion was to use border-radius property in CSS.

DEBUGGING With the Chrome DevTools

The Chrome Developer Tools (DevTools for short), are defined as a set of web authoring and debugging tools built into Google Chrome. It provide web developers deep access into the internals of the browser and their web application.

There are two ways to access the DevTools.

  1. Select the Chrome menu Chrome Menu at the top-right of your browser window, then select Tools then Developer Tools.
  2. Right-click on any page element and select Inspect Element.

Once opened, the DevTools are organised into task-oriented groups in the toolbar at the top of the window. Each toolbar item and corresponding panel let you work with a specific type of page or app information. There are eight main groups of tools available.

  • Elements (for inspection and On-The-Fly editing of the DOM elements)
  • Resources (this panel lets you inspect resources that are loaded in the inspected page. It lets you interact with HTML5 Database, Local Storage, Cookies, AppCache, etc.)
  • Network (provides insights into resources that are requested and downloaded over the network in real time, helping optimize your page.)
  • Sources
  • Timeline (gives you a complete overview of where time is spent when loading and using your web app or page.)
  • Profiles (this panel lets you profile the execution time and memory usage of a web app or page.)
  • Audits (this panel can analyze a page as it loads. Then provides suggestions and optimizations for decreasing page load time.)
  • Console (for testing Javascript web pages and applications)
